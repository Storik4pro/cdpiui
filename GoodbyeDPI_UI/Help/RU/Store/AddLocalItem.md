# Добавление элемента из локального пакета

Начиная с версии 0.3.0.0 Магазин поддерживает ручную установку и обновление элементов.

### Для чего это нужно?

Функция локальной установки элементов и обновлений может быть полезна, если:

- По какой-либо причине приложение не может получить доступ к серверу Магазина для скачивания и обновления элементов.
- Вы хотите установить старую версию элемента, которая не доступна через Магазин.
- Элемент, который вы хотите установить в настоящий момент не представлен в Магазине.
- Требуется выполнить локальное развертывание приложения как можно быстрее, игнорируя Магазин.

### Это безопасно?

Не смотря на то, что все пакеты проходят обязательную сертификацию и приложение проверяет целостность подписи пакета, не стоит терять бдительность.
Даже подписанный пакет может содержать в себе вредоносное ПО.

Устанавливайте пакет только в том случае, если вы полностью уверены в его безопасности и доверяете поставщику.

### Какие бывают виды пакетов?

- **CDPISIGNEDPACK** — стандартный пакет, обладающий подписью. Может содержать в себе надстройку, компонент или набор пресетов.
- **CDPICONFIGPACK** — пакет предназначенный для поставки наборов пресетов. Не обладает подписью. Может содержать в себе только набор пресетов и имеет ограничение на поддерживаемые типы файлов при распаковке.
- **CDPIPATCH** — пакет обновления приложения. Обладает подписью. Содержит файлы необходимые для обновления приложения.

### Как установить локальный пакет?

Установить элемент из локального пакета можно несколькими способами.

1. Перетащите пакет на ярлык приложения.
2. Кликните правой кнопкой мыши по файлу пакета и выберите [Открыть с помощью]. Затем укажите путь к исполняемому файлу приложения `CDPIUI.exe`.
3. Откройте Магазин и перейдите на страницу [Параметры]. Выберите [Добавить элемент из локального пакета]. Согласитесь с рисками.

После выполнения одного из вышеперечисленных действий откроется диалог с предложением установить элемент. Выберите [Установить] и дождитесь завершения установки.
